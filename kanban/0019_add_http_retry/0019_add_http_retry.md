# HTTPリクエストにリトライ機構を追加

## 目的
安定性を高めるため

## 要望
HTTPリクエストにリトライ機構をつけてください

## 仕様
- リトライは最大5回
- 5回連続で失敗したら例外を投げる
- エクスポーネンシャルバックオフを実装する

## 完了サマリー

**完了日時**: 2026-05-05T17:45:00+09:00

### 実装内容

`src/utils/http/interval_fetcher.py` に以下を実装:

1. **リトライロジック**
   - `make_interval_fetcher()` に `max_retries` と `initial_backoff` パラメータを追加
   - リトライ対象エラー: `ConnectionError`, `Timeout`, HTTP 5xx, HTTP 429
   - リトライ非対象: HTTP 4xx, その他の例外

2. **エクスポーネンシャルバックオフ**
   - 計算式: `wait_time = initial_backoff * (2**attempt)`
   - デフォルト: 初期バックオフ 1 秒、最大 5 回リトライ
   - 待機時間: 1秒 → 2秒 → 4秒 → 8秒 → (5回目失敗で例外)

3. **間隔制御との統合**
   - 最初の試行のみ前回リクエスト時刻からの間隔を考慮
   - リトライ試行ではエクスポーネンシャルバックオフのみで待機
   - 成功時のみ `dt_prev` を更新

4. **ログ記録**
   - リトライ試行時にログ出力（試行番号、エラー情報、待機時間）
   - 既存の `@logging_function` デコレータで自動ログ記録
